Here are some notes on the set up and some hints.

What I did to install it is in github.

The tensorflow machine learning part is in classify_image_server.py in the /home/pi/ directory, but you most likely don't need to touch that part.

You might like to edit classify_image_client.py which handles the button press and talks to the classify_image_server.py via http on port 8080

buttons.py shows you a different way of talking to the buttons. There's more information about the phatbeat library in the phatbeat docs - you can also use it to set the led pixels too, as well as the buttons.

classify-image-server.service and classify-image-client.service are just a way of making it start automatically. More details about that are at the end of the installation README file.

If you want to change it, make edits to classify_image_client.py and then restart the client:

sudo systemctl restart classify-image-client.service

To restart the server (which you probably don't want to do unless it starts saying the same thing over and over again), do:

sudo systemctl restart classify-image-server.service

you can also do sudo systemctl stop [service] or sudo systemctl start [service]

If you have trouble look in the system logs, e.g.

tail -f /var/log/syslog

and you can always reboot it: sudo reboot.

If you see stuff in the logs about the camera not being connected, check that the connector is correctly in the pi, with the blue part facing the ethernet port, and that the tiny connector on the camera has not become disconnected.
